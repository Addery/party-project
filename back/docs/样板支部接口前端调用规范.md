# 样板支部接口前端调用规范

## 1. 接口概述

本文档定义了样板支部相关接口的前端调用规范，包括获取样板支部列表和获取样板支部详情两个接口。前端开发人员可根据此规范进行接口调用，实现样板支部相关功能。

## 2. 接口列表

| 接口名称 | 接口地址 | 请求方式 | 功能描述 |
|---------|---------|---------|---------|
| 获取样板支部列表 | `/api/branches/model` | `GET` | 获取样板支部列表数据，支持分页和排序 |
| 获取样板支部详情 | `/api/branches/model/{id}` | `GET` | 根据ID获取样板支部的详细信息 |

## 3. 接口详情

### 3.1 获取样板支部列表接口

#### 接口地址
```
http://localhost:8080/api/branches/model
```

#### 请求方式
```
GET
```

#### 请求参数

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|-------|------|
| limit | Integer | 否 | 10 | 每页显示的记录数 |
| sortBy | String | 否 | "createdAt" | 排序字段，可选值："createdAt"(创建时间)、"id"(ID) |
| order | String | 否 | "desc" | 排序方式，可选值："asc"(升序)、"desc"(降序) |

## 4. 错误处理

### 4.1 错误响应格式

```json
{
  "code": 400,
  "message": "错误消息"
}
```

### 4.2 常见错误码

| 错误码 | 错误消息 | 说明 |
|-------|---------|------|
| 400 | 无效的参数 | 请求参数格式错误或参数值不合法 |
| 401 | 未授权访问 | 未登录或登录状态已过期 |
| 404 | 资源不存在 | 请求的样板支部不存在 |
| 500 | 服务器内部错误 | 服务器处理请求时发生错误 |

## 6. 注意事项

1. **接口地址**：实际部署时，将`http://localhost:8080`替换为正式环境的服务器地址。

2. **数据类型转换**：前端接收到的`createdAt`和`updatedAt`字段为ISO8601格式的字符串，可根据需要转换为Date对象。

3. **分页处理**：当数据量较大时，建议使用分页参数`limit`控制每页显示的记录数，避免一次性加载过多数据。

4. **排序处理**：根据业务需求选择合适的排序字段和排序方式，提升用户体验。

5. **错误处理**：务必处理各种错误情况，包括网络错误、服务器错误和业务逻辑错误，为用户提供友好的错误提示。

6. **图片加载**：样板支部图片使用了外部URL，前端需考虑图片加载失败的情况，提供默认图片或占位符。

7. **数据安全**：虽然这些接口允许匿名访问，但前端仍需注意数据的安全性，避免泄露敏感信息。
