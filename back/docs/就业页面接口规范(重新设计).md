# 就业页面接口规范（重新设计）

## 1. 概述

本文档定义了就业页面所需的所有API接口规范，包括接口路径、请求参数、响应格式等。

## 2. 接口列表

| 序号 | 接口名称 | 接口路径 | 请求方法 | 功能描述 |
|------|---------|---------|---------|----------|
| 1 | 获取就业统计数据 | /api/employment/stats | GET | 获取就业统计数据，包括就业率、企业数等 |
| 2 | 获取老师指导学生情况 | /api/employment/teacher-guidance | GET | 获取支部老师指导学生就业情况排名 |
| 3 | 获取就业资讯列表 | /api/employment/news | GET | 获取就业资讯列表 |
| 4 | 获取就业资讯详情 | /api/employment/news/{id} | GET | 获取就业资讯详情 |
| 5 | 获取企业招聘信息列表 | /api/employment/company-recruitment | GET | 获取企业招聘信息列表 |
| 6 | 获取就业指导资源列表 | /api/employment/guidance | GET | 获取就业指导资源列表 |
| 7 | 获取就业政策列表 | /api/employment/policies | GET | 获取就业政策列表 |
| 8 | 获取就业服务列表 | /api/employment/services | GET | 获取就业服务列表 |

## 3. 详细接口规范

### 3.1 获取就业统计数据

#### 接口路径
`/api/employment/stats`

#### 请求方法
`GET`

#### 请求参数
无

#### 响应参数
| 参数名 | 类型 | 描述 |
|-------|------|------|
| code | int | 返回码，200表示成功，其他表示失败 |
| msg | string | 返回消息 |
| data | object | 统计数据对象 |
| data.employmentRate | float | 本科生就业率 |
| data.graduateEmploymentRate | float | 研究生就业率 |
| data.companyCount | int | 合作企业数 |
| data.recruitmentCount | int | 招聘岗位数 |

#### 响应示例
```json
{
  "code": 200,
  "msg": "获取成功",
  "data": {
    "employmentRate": 97.70,
    "graduateEmploymentRate": 98.62,
    "companyCount": 300,
    "recruitmentCount": 1500
  }
}
```

### 3.2 获取老师指导学生情况

#### 接口路径
`/api/employment/teacher-guidance`

#### 请求方法
`GET`

#### 请求参数
| 参数名 | 类型 | 必填 | 描述 |
|-------|------|------|------|
| type | string | 否 | 学生类型，undergraduate(本科生)/graduate(研究生)，默认undergraduate |

#### 响应参数
| 参数名 | 类型 | 描述 |
|-------|------|------|
| code | int | 返回码，200表示成功，其他表示失败 |
| msg | string | 返回消息 |
| data | array | 老师指导学生情况列表 |
| data[].id | long | 老师ID |
| data[].name | string | 老师姓名 |
| data[].studentCount | int | 指导学生人数 |
| data[].rankChange | int | 排名变化，正数表示上升，负数表示下降，0表示不变 |

#### 响应示例
```json
{
  "code": 200,
  "msg": "获取成功",
  "data": [
    {
      "id": 1,
      "name": "张三",
      "studentCount": 25,
      "rankChange": 1
    },
    {
      "id": 2,
      "name": "李四",
      "studentCount": 22,
      "rankChange": 0
    }
  ]
}
```

### 3.3 获取就业资讯列表

#### 接口路径
`/api/employment/news`

#### 请求方法
`GET`

#### 请求参数
| 参数名 | 类型 | 必填 | 描述 |
|-------|------|------|------|
| page | int | 否 | 页码，默认1 |
| pageSize | int | 否 | 每页条数，默认10 |
| category | string | 否 | 资讯分类 |

#### 响应参数
| 参数名 | 类型 | 描述 |
|-------|------|------|
| code | int | 返回码，200表示成功，其他表示失败 |
| msg | string | 返回消息 |
| data | object | 分页数据对象 |
| data.list | array | 资讯列表 |
| data.list[].id | long | 资讯ID |
| data.list[].title | string | 资讯标题 |
| data.list[].publishDate | string | 发布日期，格式：yyyy-MM-dd HH:mm:ss |
| data.list[].source | string | 资讯来源 |
| data.list[].category | string | 资讯分类 |
| data.total | int | 总记录数 |
| data.page | int | 当前页码 |
| data.pageSize | int | 每页条数 |
| data.pages | int | 总页数 |

#### 响应示例
```json
{
  "code": 200,
  "msg": "获取成功",
  "data": {
    "list": [
      {
        "id": 1,
        "title": "2025年全国高校毕业生就业形势分析",
        "publishDate": "2025-11-20 00:00:00",
        "source": "教育部",
        "category": "就业分析"
      }
    ],
    "total": 5,
    "page": 1,
    "pageSize": 5,
    "pages": 1
  }
}
```

|-------|------|------|------|
| id | long | 是 | 资讯ID，通过路径参数传递 |

| data.viewCount | int | 浏览量 |
| data.category | string | 资讯分类 |

## 5. 错误码定义

| 错误码 | 错误信息 | 描述 |
|-------|---------|------|
| 200 | 获取成功 | 请求成功 |
| 400 | 参数错误 | 请求参数错误 |
| 404 | 资源不存在 | 请求的资源不存在 |
| 500 | 服务器错误 | 服务器内部错误 |

## 6. 接口访问权限

所有接口均允许匿名访问，无需认证。

## 7. 响应格式统一要求

所有接口的响应格式统一为JSON格式，包含code、msg、data三个字段：

```json
{
  "code": 200,
  "msg": "获取成功",
  "data": {}
}
```
