# 新的用户画像接口说明

## 1. 接口概述

本次修改主要针对用户画像接口的认证和授权机制进行了优化，具体包括：

1. **修改了JWT令牌的生成格式**：现在令牌包含了用户名和用户ID信息
2. **优化了获取当前用户画像的逻辑**：从JWT令牌中直接提取用户ID，不再依赖SecurityContextHolder
3. **保持了API接口路径不变**：确保与现有调用代码的兼容性

## 2. JWT令牌格式变化

### 2.1 原令牌格式
```
fake-jwt-token-username
```

### 2.2 新令牌格式
```
fake-jwt-token-userId-username
```

例如：
```
fake-jwt-token-6-Addery
```

## 4. 实现细节说明

### 4.1 JWT令牌的生成与解析

**生成令牌**：
```java
// 在UserService.login方法中
token = jwtUtils.generateToken(user.getUsername(), user.getId());
```

**解析令牌**：
```java
// 从Authorization头中提取token
String token = authorizationHeader.substring(7);

// 验证token
if (!jwtUtils.validateToken(token)) {
    return Response.error("令牌无效");
}

// 从token中获取用户ID
Integer userId = jwtUtils.getUserIdFromToken(token);

// 从token中获取用户名
String username = jwtUtils.getUsernameFromToken(token);
```

### 4.2 从token中提取用户ID的逻辑

```java
// 在JwtUtils类中
public Integer getUserIdFromToken(String token) {
    if (token != null && token.startsWith("fake-jwt-token-")) {
        String[] parts = token.substring(15).split("-");
        if (parts.length >= 1) {
            try {
                return Integer.parseInt(parts[0]);
            } catch (NumberFormatException e) {
                return null;
            }
        }
    }
    return null;
}
```
