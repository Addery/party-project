# 注册接口对接规范

## 1. 接口概述

该接口用于新用户注册，实现了白名单机制，只有白名单内的手机号+身份证号组合才能进行注册。

## 2. 接口详情

### 2.1 接口URL
```
POST /api/auth/register
```

### 2.2 请求方法
```
POST
```

### 2.3 请求头
```
Content-Type: application/json
```

### 2.4 请求参数

| 参数名 | 类型 | 必填 | 描述 | 验证规则 |
|--------|------|------|------|----------|
| username | String | 是 | 用户名 | 长度3-50个字符 |
| password | String | 是 | 密码 | 长度6-20个字符，包含大小写字母和数字 |
| name | String | 是 | 真实姓名 | 不能为空 |
| gender | String | 否 | 性别 | 可选值：男、女 |
| age | Integer | 否 | 年龄 | 18-100岁 |
| partyStatus | String | 否 | 党籍状态 | 可选值：入党申请人、入党积极分子、发展对象、预备党员、正式党员 |
| branch | String | 否 | 所属支部 | 默认值："默认支部" |
| phone | String | 是 | 手机号码 | 格式：1开头11位数字 |
| email | String | 否 | 邮箱 | 符合邮箱格式 |
| idCard | String | 是 | 身份证号 | 18位，符合身份证号格式 |
| joinDate | String | 否 | 入党日期 | 格式：YYYY-MM-DD，不能是未来日期 |
| position | String | 否 | 职位 | - |
| education | String | 否 | 学历 | 可选值：初中、高中、大专、本科、硕士、博士 |
| workYears | Integer | 否 | 工作年限 | 大于等于0 |
| avatar | String | 否 | 头像URL | - |
| role | String | 否 | 用户角色 | 可选值：member、admin，默认值：member |

## 3. 白名单机制

注册接口实现了白名单验证机制：

1. 系统会检查请求中的`phone`和`idCard`组合是否在白名单中
2. 只有在白名单中的组合才能成功注册
3. 白名单数据存储在`registration_whitelist`表中，包含以下字段：
   - id: 主键
   - phone: 手机号码
   - id_card: 身份证号
   - remark: 备注信息
4. 白名单表中有唯一约束：`uk_phone_idcard (phone, id_card)`

## 4. 错误码说明

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| 400 | 该手机号和身份证号组合不在注册白名单中 | 注册的手机号+身份证号组合未在白名单中 |
| 400 | 注册需要提供手机号和身份证号 | 未提供手机号或身份证号 |
| 400 | 用户名已存在 | 用户名已被其他用户使用 |
| 400 | 邮箱已被注册 | 邮箱已被其他用户使用 |
| 400 | 手机号已被注册 | 手机号已被其他用户使用 |
| 400 | 密码不符合要求：密码长度必须在6-20个字符之间，包含大小写字母和数字 | 密码强度不足 |
| 400 | 手机号码格式不正确 | 手机号格式不符合要求 |
| 400 | 身份证号格式不正确 | 身份证号格式不符合要求 |
| 400 | 邮箱格式不正确 | 邮箱格式不符合要求 |
| 400 | 年龄不能小于18岁 | 年龄不符合要求 |
| 400 | 年龄不能大于100岁 | 年龄不符合要求 |
| 400 | 入党日期不能是未来日期 | 入党日期不符合要求 |

## 6. 注意事项

1. 密码需要符合强度要求：长度6-20个字符，包含大小写字母和数字
2. 手机号和身份证号必须同时提供，且组合必须在白名单中
3. 用户名、邮箱、手机号具有唯一性约束
4. 入党日期不能是未来日期
5. 所有日期格式为YYYY-MM-DD
6. 响应中的敏感信息（如密码）会被隐藏
