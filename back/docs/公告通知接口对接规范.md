# 公告通知接口对接规范

## 1. 接口概述

本文档定义了公告通知相关的接口规范，包括接口URL、请求方法、参数说明、响应数据结构等，用于前后端对接开发。

## 2. 接口列表

| 接口名称 | 接口URL | 请求方法 | 功能描述 |
| --- | --- | --- | --- |
| 获取首页公告 | `/api/notices/home` | GET | 获取首页显示的最新通知公告列表 |
| 获取公告列表 | `/api/notices` | GET | 获取分页的通知公告列表 |
| 获取公告详情 | `/api/notices/{id}` | GET | 获取指定通知公告的详细信息 |
| 获取公告分类 | `/api/notices/categories` | GET | 获取通知公告分类列表 |

## 3. 接口详细说明

### 3.1 获取首页公告

**接口URL**：`/api/notices/home`

**请求方法**：GET

**请求参数**：

| 参数名称 | 参数类型 | 是否必填 | 默认值 | 参数说明 |
| --- | --- | --- | --- | --- |
| limit | Integer | 否 | 5 | 返回的通知公告数量 |

**响应数据结构**：

```json
{
  "code": 200,
  "message": "获取成功",
  "data": [
    {
      "id": 1,
      "title": "关于召开全体党员大会的通知",
      "date": "2025-12-10",
      "department": "党委办公室"
    },
    {
      "id": 2,
      "title": "2025年度优秀党员评选结果公示",
      "date": "2025-12-05",
      "department": "组织部"
    }
  ]
}
```

**响应参数说明**：

| 参数名称 | 参数类型 | 参数说明 |
| --- | --- | --- |
| id | Integer | 公告ID |
| title | String | 公告标题 |
| date | String | 发布日期，格式：yyyy-MM-dd |
| department | String | 发布部门 |

### 3.2 获取公告列表

**接口URL**：`/api/notices`

**请求方法**：GET

**请求参数**：

| 参数名称 | 参数类型 | 是否必填 | 默认值 | 参数说明 |
| --- | --- | --- | --- | --- |
| page | Integer | 否 | 1 | 页码 |
| pageSize | Integer | 否 | 10 | 每页数量，最大50 |
| categoryId | Integer | 否 | 无 | 通知分类ID（暂未实现） |
| keyword | String | 否 | 无 | 搜索关键词，用于模糊搜索标题和内容 |

**响应数据结构**：

```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "total": 100,
    "page": 1,
    "pageSize": 10,
    "pages": 10,
    "notices": [
      {
        "id": 1,
        "title": "关于召开全体党员大会的通知",
        "category": "会议通知",
        "publishDate": "2025-12-10",
        "source": "党委办公室"
      },
      {
        "id": 2,
        "title": "2025年度优秀党员评选结果公示",
        "category": "公示公告",
        "publishDate": "2025-12-05",
        "source": "组织部"
      }
    ]
  }
}
```

**响应参数说明**：

| 参数名称 | 参数类型 | 参数说明 |
| --- | --- | --- |
| total | Long | 公告总数 |
| page | Integer | 当前页码 |
| pageSize | Integer | 每页数量 |
| pages | Integer | 总页数 |
| notices | List<NoticeItemDTO> | 公告列表数据 |
| └ id | Integer | 公告ID |
| └ title | String | 公告标题 |
| └ category | String | 公告分类 |
| └ publishDate | String | 发布日期，格式：yyyy-MM-dd |
| └ source | String | 发布部门 |

### 3.3 获取公告详情

**接口URL**：`/api/notices/{id}`

**请求方法**：GET

**请求参数**：

| 参数名称 | 参数类型 | 是否必填 | 默认值 | 参数说明 |
| --- | --- | --- | --- | --- |
| id | Integer | 是 | 无 | 公告ID，通过URL路径传递 |

**响应数据结构**：

```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "id": 1,
    "title": "关于召开全体党员大会的通知",
    "category": "会议通知",
    "publishDate": "2025-12-10",
    "source": "党委办公室",
    "content": "<p>全体党员同志：</p><p>兹定于2025年12月15日上午9:00在大礼堂召开全体党员大会，请各位党员准时参加。</p>",
    "attachments": []
  }
}
```

**响应参数说明**：

| 参数名称 | 参数类型 | 参数说明 |
| --- | --- | --- |
| id | Integer | 公告ID |
| title | String | 公告标题 |
| category | String | 公告分类 |
| publishDate | String | 发布日期，格式：yyyy-MM-dd |
| source | String | 发布部门 |
| content | String | 公告内容，HTML格式 |
| attachments | List<AttachmentDTO> | 附件列表（目前暂未实现，返回空列表） |

### 3.4 获取公告分类

**接口URL**：`/api/notices/categories`

**请求方法**：GET

**请求参数**：无

**响应数据结构**：

```json
{
  "code": 200,
  "message": "获取成功",
  "data": [
    "会议通知",
    "公示公告",
    "政策文件",
    "党建工作"
  ]
}
```

**响应参数说明**：

| 参数名称 | 参数类型 | 参数说明 |
| --- | --- | --- |
| data | List<String> | 公告分类列表 |

## 5. 对接注意事项

1. **接口认证**：
   - 公告通知接口目前无需认证，所有用户均可访问
   - 若后续接口增加认证要求，将另行通知

2. **参数说明**：
   - 所有可选参数如未提供，将使用默认值
   - 分页参数中，`page`最小值为1，`pageSize`最大值为50

3. **响应处理**：
   - 接口返回统一的JSON格式，包含`code`、`message`和`data`字段
   - 应先检查`code`字段是否为200，再处理`data`字段
   - 若`code`不为200，应根据`message`字段提示用户错误信息

4. **日期格式**：
   - 所有日期字段格式为`yyyy-MM-dd`
   - 前端可根据需要进行日期格式化显示

5. **内容处理**：
   - 公告内容为HTML格式，前端需使用富文本编辑器或`dangerouslySetInnerHTML`等方式安全渲染
   - 建议对HTML内容进行过滤，防止XSS攻击

6. **附件功能**：
   - 当前版本附件功能暂未实现，`attachments`字段返回空列表
   - 若后续实现附件功能，将保持接口兼容性，仅扩展`attachments`字段的内容

7. **分类功能**：
   - 当前版本支持获取公告分类列表，但`categoryId`参数暂未实现过滤功能
   - 若后续实现分类过滤，将保持接口兼容性
