## 3. 现有接口优化建议

### 3.1 增加分页和筛选功能

部分现有接口需要增加分页和筛选功能，例如：

1. **获取就业服务列表**：
   - 当前接口：`/api/employment/services`
   - 优化建议：支持分页参数（page, pageSize）和筛选参数（serviceName, serviceType, serviceObject）

2. **获取就业政策列表**：
   - 当前接口：`/api/employment/policies`
   - 优化建议：确保支持分页参数（page, pageSize）

### 3.2 增加排序功能

部分接口需要增加排序功能，例如：

1. **获取就业指导资源列表**：
   - 当前接口：`/api/employment/guidance`
   - 优化建议：支持sortBy参数（如viewCount, publishDate）和sortOrder参数（asc, desc）

2. **获取企业招聘信息列表**：
   - 当前接口：`/api/employment/recruitment`
   - 优化建议：支持sortBy参数（如publishDate, salary）和sortOrder参数（asc, desc）

## 4. 接口返回数据格式建议

为了保持前后端数据格式的一致性，建议所有接口返回以下格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [],          // 数据列表
    "total": 0,          // 总记录数
    "pages": 0,          // 总页数
    "current": 1,        // 当前页码
    "size": 10           // 每页数量
  }
}
```

对于不需要分页的接口（如热门资源、最新资源等），可以返回简化格式：

```json
{
  "code": 200,
  "message": "success",
  "data": [
    // 数据列表
  ]
}
```
